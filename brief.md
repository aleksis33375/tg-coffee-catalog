# Brief: Каталог услуг кофейни — Telegram Mini App

> Версия: 1.0  
> Дата: апрель 2026  
> Основа: research.md + экспертная оценка

---

## Концепция одной строкой

Клиент открывает Telegram-бота → нажимает кнопку → видит услуги с ценами → нажимает «Выбрать» → попадает в чат с менеджером с готовым текстом заявки.

---

## Принципы, которые не нарушаем

- **Открытие = сразу каталог.** Без экранов-заставок, без «добро пожаловать».
- **Цена всегда видна.** Никакого «уточняйте цену у менеджера».
- **Один главный CTA на экран.** Не три кнопки — одна, большая, нативная.
- **Выглядит как приложение, не как сайт.** `tg.expand()` при старте, `BackButton`, `MainButton`.
- **Минимум шагов.** Максимум — 2 уровня навигации: список категорий → услуги.

## Экраны

---

### Экран 1 — Каталог (главный, стартовый)

**Что видит пользователь:**

```
┌─────────────────────────────┐
│  ← [BackButton Telegram]    │  ← нативная кнопка, скрыта на этом экране
├─────────────────────────────┤
│  [Логотип / название кофейни]│
│  [Короткий слоган — 1 строка]│
├─────────────────────────────┤
│  [Таб: Кофе] [Таб: Чай] ... │  ← горизонтальный скролл
├─────────────────────────────┤
│  ┌─────────┐  ┌─────────┐  │
│  │  фото   │  │  фото   │  │
│  │ Название│  │ Название│  │
│  │  500 ₽  │  │  350 ₽  │  │
│  │[Выбрать]│  │[Выбрать]│  │
│  └─────────┘  └─────────┘  │
│  ┌─────────┐  ┌─────────┐  │
│  │  ...    │  │  ...    │  │
│  └─────────┘  └─────────┘  │
└─────────────────────────────┘
```

**Элементы:**

| Элемент | Описание |
|---|---|
| Логотип + название | Фото кофейни или лого, название, слоган 1 строкой |
| Табы категорий | Горизонтальный скролл: Кофе / Чай / Альтернатива / Десерты / Всё меню |
| Карточка услуги | Фото (квадрат 1:1), название, цена, кнопка «Выбрать» |
| Бейдж «Хит» / «Новинка» | Плашка поверх фото на выбранных карточках |

**Поведение:**

- `tg.expand()` — разворачивает на весь экран сразу при открытии
- Активный таб подчёркнут / выделен цветом
- Карточки в 2 колонки, фото квадратное
- Прокрутка вертикальная, табы горизонтально
- `BackButton` скрыт — это главный экран

**Действия:**

- Нажать таб → фильтрует карточки (без перехода на новый экран)
- Нажать «Выбрать» → переход на Экран 2 (карточка услуги)

### Экран 2 — Карточка услуги

**Что видит пользователь:**

```
┌─────────────────────────────┐
│ ←  [BackButton Telegram]    │  ← нативная, ведёт назад в каталог
├─────────────────────────────┤
│                             │
│      [Фото услуги]          │  ← 16:9 или квадрат, на всю ширину
│                             │
├─────────────────────────────┤
│  Название услуги            │  ← крупно
│  500 ₽                      │  ← цена сразу под названием
├─────────────────────────────┤
│  Описание — 2–3 строки      │  ← что входит, сколько времени
│  максимум, без воды         │
├─────────────────────────────┤
│  ★★★★★  «Отличный кофе»     │  ← один отзыв или рейтинг (если есть)
├─────────────────────────────┤
│                             │
│  [████ Записаться █████]    │  ← Telegram MainButton, нативная
└─────────────────────────────┘
```

**Элементы:**

| Элемент | Описание |
|---|---|
| Фото | На всю ширину, 16:9 или квадрат |
| Название | Крупный шрифт, 20–22px |
| Цена | Под названием, выделена цветом акцента |
| Описание | 2–3 строки: что включено, сколько по времени |
| Отзыв | Одна звёздная оценка + короткая цитата (опционально) |
| MainButton | «Записаться» — нативная кнопка Telegram снизу |

**Поведение:**

- `tg.BackButton.show()` — нативная кнопка «назад» в хедере Telegram
- `tg.MainButton.show()` с текстом «Записаться»
- `tg.HapticFeedback.impactOccurred('medium')` при нажатии MainButton

**Действия:**

- `BackButton` → возврат на Экран 1 (каталог)
- `MainButton «Записаться»` → Экран 3 (подтверждение) или сразу в чат

### Экран 3 — Подтверждение / переход в чат

**Два варианта реализации (выбираем при разработке):**

**Вариант А — мгновенный переход (проще):**

При нажатии «Записаться» — сразу:
```js
tg.openTelegramLink('https://t.me/username?text=Хочу+записаться+на+[Название+услуги]')
```
Открывается чат с менеджером, текст заявки уже вставлен. Mini App закрывается.

**Вариант Б — экран подтверждения (сложнее, лучше UX):**

```
┌─────────────────────────────┐
│ ←  [BackButton Telegram]    │
├─────────────────────────────┤
│                             │
│      ✓  (анимация)          │  ← Lottie или CSS-анимация
│                             │
│   Отлично! Ты выбрал:       │
│   [Название услуги]         │
│   500 ₽                     │
│                             │
│   Менеджер ответит          │
│   в течение 5 минут         │
│                             │
├─────────────────────────────┤
│  [████ Написать в Telegram █]│  ← MainButton
└─────────────────────────────┘
```

**Рекомендация:** Начать с Варианта А — меньше кода, тот же результат.

## Навигационная схема

```
[Telegram-бот]
      │
      │  нажать кнопку в боте
      ▼
[Экран 1: Каталог] ──── таб Кофе / Чай / ... ──── (без перехода, фильтр)
      │
      │  нажать «Выбрать» на карточке
      ▼
[Экран 2: Карточка услуги]
      │                    │
      │ BackButton          │ MainButton «Записаться»
      ▼                    ▼
[Экран 1: Каталог]    [Чат с менеджером в Telegram]
                       (Mini App закрывается)
```

**Максимальная глубина:** 2 уровня. Пользователь никогда не теряется.

## Дизайн-система

### Цвета

Используем CSS-переменные Telegram как основу, накладываем фирменные акценты:

```css
/* Базовые — из темы Telegram */
--bg:        var(--tg-theme-bg-color);
--text:      var(--tg-theme-text-color);
--hint:      var(--tg-theme-hint-color);
--section-bg: var(--tg-theme-section-bg-color);

/* Акценты кофейни — поверх */
--accent:    #C8813A;   /* тёплый кофейный */
--accent-2:  #6B3F1F;   /* тёмный эспрессо */
--price:     #2D9E6B;   /* зелёный — цена, позитив */
--badge-hit: #E05C3A;   /* красный — «Хит» */
--badge-new: #3A7AE0;   /* синий — «Новинка» */
```

### Типографика

```css
/* Системный стек — без внешних шрифтов */
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;

/* Размеры */
--text-xl:   22px;   /* название на карточке */
--text-lg:   18px;   /* цена, заголовки */
--text-md:   15px;   /* описание, табы */
--text-sm:   13px;   /* подсказки, метки */
```

### Кнопки

```css
/* Все кнопки — минимум 44px высотой */
.btn {
  min-height: 44px;
  border-radius: 12px;
  font-size: 16px;
  font-weight: 600;
  padding: 0 20px;
}

/* Кнопка «Выбрать» на карточке */
.btn-select {
  background: var(--accent);
  color: #fff;
  width: 100%;
}
```

### Карточка услуги

```css
.card {
  border-radius: 16px;
  overflow: hidden;
  background: var(--section-bg);
  /* без border — Telegram сам добавляет разделители */
}

.card-img {
  aspect-ratio: 1 / 1;
  width: 100%;
  object-fit: cover;
}
```

## Технический стек

| Слой | Решение | Почему |
|---|---|---|
| Frontend | HTML + CSS + Vanilla JS | Нет бандлера, нет npm, быстрый старт |
| Telegram API | `window.Telegram.WebApp` | Нативный SDK, без обёрток |
| Хостинг | GitHub Pages или любой статик | Бесплатно, HTTPS, мгновенно |
| Данные | JSON-файл в репозитории | Нет базы, нет сервера — обновлять руками |
| Бот | @BotFather + webhooks (позже) | В v1 — просто ссылка на Mini App |

**Инициализация (первые строки JS):**

```js
const tg = window.Telegram.WebApp;
tg.ready();
tg.expand(); // сразу на весь экран
```

## Чего НЕ будет в версии 1.0

| Функция | Почему откладываем |
|---|---|
| Корзина / множественный выбор | В v1 — одна услуга, один контакт. Корзина усложняет. |
| Онлайн-оплата | Требует юрлицо, эквайринг, интеграцию. |
| Регистрация / профиль | Авторизация через Telegram и так есть |
| Программа лояльности | Нет базы клиентов для старта |
| Поиск по каталогу | Мало позиций, табы справятся |
| Отзывы / рейтинги | Нет механизма сбора — статический текст максимум |
| Уведомления / push | Нужен бэкенд и подписка |
| Adminка | Редактируем JSON вручную |
| Доставка / адрес | Кофейня = самовывоз |
| Мультиязычность | Только русский |
| Бронирование времени | Запись через менеджера в чате |

## Версия 2.0 — Продукт под ключ

> Апрель 2026. Надстройка над v1.0 — Mini App остаётся, добавляется бэкенд, adminка и интерфейс бариста.

---

### Что изменилось и почему

v1.0 решила задачу витрины: клиент видит меню и пишет менеджеру.  
v2.0 решает задачу бизнеса: владелец управляет кофейней, бариста работает со стойки, клиент накапливает лояльность.

| Было в v1.0 | Стало в v2.0 |
|---|---|
| Нет базы, нет сервера | SQLite + Node.js + Express на Railway |
| JSON обновляется вручную | Adminка управляет всем контентом |
| Один менеджер принимает заявки | Бариста видит заказы в реальном времени |
| Нет учёта клиентов | База покупателей с историей и источниками |
| Нет лояльности | Акции, кружки, скидки — бариста отмечает кнопкой |
| GitHub Pages | Vercel (Mini App) + Railway (бэкенд + adminка + бариста) |

---

### Три роли

| Роль | Вход | Что делает |
|---|---|---|
| Покупатель | Ссылка / QR / Telegram-бот | Смотрит меню, оформляет заказ |
| Бариста | PIN-код на планшете | Принимает заказы, отмечает акции, закрывает смену |
| Владелец | Пароль в браузере | Управляет всем: меню, аналитика, рассылки, настройки |

---

### Новый технический стек

| Слой | Решение |
|---|---|
| Mini App (покупатель) | HTML + CSS + Vanilla JS — без изменений, Vercel |
| Бэкенд | Node.js + Express, Railway |
| База данных | SQLite (better-sqlite3) |
| Adminка и бариста | HTML + CSS + JS, раздаёт Express |
| Telegram Bot | node-telegram-bot-api, webhook |
| Аналитика | Chart.js (CDN, без npm) |

### Что появилось в v2.0

**Для покупателя:**
- Регистрация при первом открытии — попадает в базу автоматически

**Для владельца:**
- Центральный дашборд: выручка, средний чек, топ позиций, источники трафика, воронка
- Клик на график → детальная таблица без перехода на другой экран
- Управление меню: цены, фото, скрыть позицию без удаления
- Акции любого типа — создаёт сам без разработчика
- Рассылки по сегментам: все / покупатели / давно не заходили / VIP
- Автосообщения: приветствие, день рождения, «давно не заходил»
- Список сотрудников: каждый бариста со своим PIN
