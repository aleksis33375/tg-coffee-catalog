# TESTING.md — Руководство тестировщика
## Hot Black Coffee — Telegram Mini App

---

## 1. Что это за проект

Telegram Mini App для кофейни Hot Black Coffee. Клиент открывает бота, видит меню,
добавляет позиции в корзину и оформляет заказ — заявка уходит в Telegram-чат менеджеру.

Бэкенда нет. Всё работает как статический сайт: HTML + CSS + JS + menu.json.

## 3. Как запустить

### Способ А — в Telegram (основной, обязательный)

1. Открыть Telegram, найти бота **@Prototip_Coffee_house_bot**
2. Нажать **Старт**
3. Нажать кнопку **☕ Открыть меню** внизу чата
4. Mini App откроется внутри Telegram

### Способ Б — в браузере (для UI-тестирования)

Открыть в браузере: `https://tg-coffee-catalog.vercel.app`

В браузере автоматически появятся вспомогательные кнопки для навигации —
стрелка «←» (назад) и кнопка действия внизу. Это заглушки вместо нативных
кнопок Telegram. Функции openTelegramLink и HapticFeedback недоступны.

### Способ В — локально

```bash
git clone https://github.com/aleksis33375/tg-coffee-catalog.git
cd tg-coffee-catalog/tg-app
# открыть index.html в браузере (через Live Server или двойным кликом)
```

## 6. Тест-кейсы

### TC-01 — Первый визит: онбординг

**Условие:** приложение открывается впервые (localStorage пуст)

**Шаги:**
1. Открыть приложение
2. Должно появиться модальное окно онбординга

**Ожидаемый результат:**
- Показывается окно с текстом «Привет, [имя из Telegram]!»
- Если имя недоступно — «Привет, друг!»
- Три пункта: «Смотри меню и выбирай напитки», «Добавляй в корзину», «Без звонков, без очередей»
- Кнопка «Начать»
- Нажать «Начать» → онбординг плавно скрывается (fade 0.25s)
- Сразу после — показывается окно оффера

---

### TC-02 — Первый визит: оффер

**Условие:** онбординг закрыт, оффер не показывался

**Ожидаемый результат:**
- Появляется окно «Скидка 10% на первый заказ»
- Кнопка «Получить скидку 10%» → открывается бот @Prototip_Coffee_house_bot (через tg.openTelegramLink или новая вкладка), окно закрывается
- Кнопка «Пропустить» → окно закрывается
- **При следующем открытии приложения — онбординг и оффер не показываются**

---

### TC-03 — Повторный визит

**Условие:** localStorage содержит ключи `hotblack_onboarding_seen=1` и `hotblack_offer_seen=1`

**Ожидаемый результат:**
- Приложение открывается сразу на каталоге, без модальных окон

### TC-04 — Навигация по табам

**Шаги:**
1. Открыть каталог
2. Нажать таб «Кофе» → должны показаться ID 1, 2, 3, 4
3. Нажать таб «Чай» → должны показаться ID 5, 6, 8, 10
4. Нажать таб «Десерты» → должны показаться ID 7, 9, 11, 12
5. Нажать таб «Акции» → должен показаться блок с акциями (не карточки)

**Ожидаемый результат:**
- При смене таба показываются только позиции нужной категории
- Активный таб визуально выделен
- Таб «Акции» отображает два блока: «Скидка 10%» и «Каждая 6-я кружка бесплатно»

---

### TC-05 — Открытие карточки товара (Экран 2)

**Шаги:**
1. Нажать кнопку «В корзину» или саму карточку
2. Должен открыться Экран 2

**Ожидаемый результат:**
- Фото товара на всю ширину (или градиент, если фото нет)
- Название, цена, описание
- Если `badge` заполнен — плашка «хит» или «новинка» поверх фото
- Степпер «−» / количество / «+»
- Кнопка Telegram MainButton с текстом «← В меню»
- Нативная BackButton Telegram в верхнем левом углу (только в Telegram)

---

### TC-06 — Корзина: добавление и изменение

**Шаги:**
1. На карточке нажать «В корзину» → кнопка меняется на степпер «− 1 +»
2. Нажать «+» несколько раз → счётчик растёт
3. Нажать «−» до 0 → степпер меняется обратно на «В корзину»
4. Добавить несколько разных товаров

**Ожидаемый результат:**
- Внизу экрана появляется плавающая полоса корзины: иконка 🛒, количество позиций, общая сумма
- Сумма пересчитывается при каждом изменении
- При 0 товаров полоса корзины скрыта

---

### TC-07 — Оформление заказа (Экран 3)

**Шаги:**
1. Добавить минимум 1 товар в корзину
2. Нажать кнопку «Оформить →» в полосе корзины
3. Должен открыться Экран 3

**Ожидаемый результат:**
- Анимированная галочка (SVG-анимация)
- Список позиций: название, количество, сумма за позицию
- Строка «Итого: X ₽»
- Переключатель «🏠 Самовывоз» / «🚗 Доставка»
- При «Самовывоз»: поле «К какому времени приготовить?» (time input), поле адреса скрыто
- При «Доставка»: появляется поле «Адрес доставки»
- Кнопка MainButton «Написать в Telegram»

### TC-10 — Программа лояльности (Акции → кружки)

**Шаги:**
1. Перейти на таб «Акции»
2. Найти блок «Каждая 6-я кружка бесплатно»
3. Нажать «☕ Отметить кружку» 5 раз

**Ожидаемый результат:**
- При каждом нажатии заполняется одна из 6 кружек
- После 5 нажатий кнопка меняется на «🎁 Отметить бесплатную»
- После 6-го нажатия: кнопка «✓ Готово», статус «🎉 Готово! Покажи этот экран бариста»
- Нажать «↺» → появляется confirm «Сбросить счётчик кружек?» → ОК → счётчик обнуляется
- Данные сохраняются в localStorage (`hotblack_cups_count`) и не теряются при перезагрузке

## 7. Сброс состояния для повторного теста

Для повторного тестирования первого визита нужно очистить localStorage.

**В браузере:**
- DevTools → Application → Local Storage → очистить все ключи `hotblack_*`

**В Telegram (через DevTools в десктопном Telegram):**
- Меню → Settings → Advanced → Experimental Settings → Enable webview inspecting
- Открыть Mini App → правая кнопка → Inspect → Console:
```js
localStorage.removeItem('hotblack_onboarding_seen');
localStorage.removeItem('hotblack_offer_seen');
localStorage.removeItem('hotblack_cups_count');
```

## 8. Устройства для тестирования

| Устройство | Приоритет | Что проверять дополнительно |
|---|---|---|
| iPhone (iOS 15+), Telegram | Критично | safe-area (чёлка/Dynamic Island), нативные кнопки |
| Android (любой), Telegram | Критично | кнопки не обрезаны снизу, шрифты |
| Telegram Desktop (Windows/Mac) | Важно | планшетный вид 768px, центрирование |
| Chrome / Safari на ПК | Дополнительно | браузерный fallback, общий вид |

---

## 9. Известные ограничения

- `tg.HapticFeedback` и `tg.BackButton` работают **только внутри Telegram**, в браузере недоступны
- Кнопка «Открыть в Telegram» в браузере использует `window.open` вместо `tg.openTelegramLink`
- Фото товаров загружаются из папки `img/` — при локальном запуске нужен Live Server (двойной клик на HTML даст ошибку CORS для JSON)
- Программа лояльности хранится локально в браузере пользователя — сброс при очистке данных
