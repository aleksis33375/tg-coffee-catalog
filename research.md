# Исследование: Telegram Mini Apps для кофеен и кафе

> Дата: апрель 2026  
> Цель: понять рынок перед разработкой каталога услуг кофейни как TMA

---

## 5 примеров Telegram Mini App

---

### 1. Simple Apps — TMA для кофейни

**Что делает:** Бесплатный предзаказ с самовывозом, разгрузка очередей в час пик.

**Функционал:**
- Каталог: напитки, выпечка, снеки с фото и описаниями
- Выбор временного слота (интервалы 15 минут)
- Кастомизация напитка: молоко, сахар, размер стакана
- Upsell-рекомендации на шаге корзины
- Push-уведомления о готовности заказа
- Программа лояльности + реферальная механика
- Админ-панель: меню, склад, лимиты на слоты, индикатор загрузки

**Результаты у клиента:** −40% очередей, +25% средний чек, 60% клиентов перешли в бот.

**Сильные стороны:**
- Система временных слотов решает реальную операционную проблему
- Upsell встроен органично, без навязчивости
- Индикатор загрузки по слотам — редкость даже среди SaaS-решений
- Авторизация через Telegram — не нужна регистрация

**Слабые стороны:**
- Индивидуальная разработка, нет публичного демо
- Высокая плотность функций — может ощущаться перегруженно на мобиле
- Нет информации о поддержке тёмной/светлой темы

**Что взять:** идею временных слотов и upsell на шаге корзины.

### 2. Cafebotum — SaaS-платформа для кафе

**Что делает:** Конструктор TMA для кафе, ресторанов, суши-баров. Запуск за 7 минут, без комиссии с заказов.

**Функционал:**
- Онлайн-меню с модификаторами и категориями
- Корзина и оформление заказа внутри Telegram
- Программа лояльности с гибкими настройками
- Промокоды и сегментация клиентов
- Несколько локаций
- Аналитика: выручка, заказы, средний чек, RFM
- Рассылки (текст, фото, видео, голос)
- Опциональный лендинг

**Цены:** Бесплатно (30 позиций) / ₽2 490/год (100 позиций) / ₽3 490/год (1 000 позиций + доставка + оплата).

**Сильные стороны:**
- Бесплатный тариф реально рабочий (30 позиций хватит небольшой кофейне)
- RFM-анализ — серьёзный инструмент за небольшие деньги
- Голосовые/видео-рассылки внутри Telegram — нестандартно и вовлекающе
- Модель без комиссии выгодна при росте объёма

**Слабые стороны:**
- На сайте нет скриншотов клиентского интерфейса — непонятно как выглядит
- Лимит 30 позиций на бесплатном тарифе ограничивает полноценное кафе
- Инструменты сегментации могут быть сложны для небольших операторов
- Нет прямой ссылки на демо-бот

**Что взять:** структуру категорий, принцип без-комиссионной модели как аргумент для клиента.

### 3. Smartofood — SaaS с интеграцией iiko

**Что делает:** Корпоративное решение для ресторанов на iiko: меню синхронизируется из кассы автоматически.

**Функционал:**
- Доставка и самовывоз
- Автосинхронизация меню из iiko (стоп-листы в реальном времени)
- Интеграция iikoCard (бонусы работают без переподписки)
- Онлайн-оплата с фискальным чеком
- Уведомления о статусе заказа
- Промокоды
- Отзывы + чаевые
- Управление зонами доставки
- Яндекс.Метрика

**Цены:** от ₽3 900/месяц.

**Сильные стороны:**
- Синхронизация iiko — главный дифференциатор: стоп-листы обновляются сами
- Интеграция iikoCard без переподписки клиентов
- Фискальные чеки — юридически важно в России

**Слабые стороны:**
- ₽3 900/мес — дорого для маленькой кофейни
- Весь смысл пропадает без iiko
- Нет бесплатного тарифа

**Что взять:** идею отображения стоп-листов и статусов заказа в реальном времени.

### 4. tma-cafe — Open-source демо «Laurel Cafe»

**Что делает:** Учебный open-source проект на GitHub, справочная реализация базовых паттернов TMA.

**Функционал:**
- Главная: информация о кафе + популярные блюда
- Каталог по категориям
- Карточка товара с выбором вариантов
- Корзина с редактированием
- Интеграция Telegram Payments
- Корзина сохраняется между сессиями (localStorage)

**Стек:** Vanilla JS + HTML/CSS + jQuery, Lottie-анимации, Material Symbols, Rubik font, Flask backend.

**Сильные стороны:**
- Vanilla-стек — легко читать и форкать
- Lottie-анимации для переходов (успех, загрузка) — ощущается нативно
- Корзина сохраняется между сессиями
- Чёткое разделение информационных и транзакционных экранов

**Слабые стороны:**
- Не production-ready, нет админ-панели
- Material Symbols — выглядит холодно для кофейной эстетики
- jQuery — устарел, сложно расширять
- Нет лояльности, уведомлений, истории заказов

**Что взять:** архитектуру экранов, паттерн сохранения корзины, подход к Lottie-анимациям.

### 5. tgEDA — SaaS для ресторанов (800+ локаций)

**Что делает:** Одна из первых крупных платформ TMA для общепита в России. Сейчас переходит в экосистему MAX (VK).

**Функционал:**
- Меню с привязкой ко времени суток (завтраки — с 8:00, обеды — с 12:00)
- Онлайн-оплата
- Доставка и самовывоз
- Промо-инструменты
- Рассылки клиентам
- Бронирование стола (есть, но почти не используется)
- Сбор контактов перед показом меню

**Реальные данные операторов:**
- 150–800 заказов в месяц на одну точку
- 97% клиентов дают скидку или подарок на первый заказ
- Клиенты терпят 2–4 промо-рассылки в месяц без отписки
- 92% ресторанов собирают контакты перед показом меню

**Сильные стороны:**
- Меню по времени суток — умная фича для мест с разными меню
- Сбор контактов перед каталогом — мощный CRM-инструмент
- 800 локаций — реально рабочее решение, не маркетинг

**Слабые стороны:**
- Переходит в MAX — неясно будущее в Telegram
- Бронирование стола есть, но операторы не используют (UX слишком сложный?)
- Нет публичных скриншотов клиентского интерфейса

**Что взять:** идею временно́го меню и мягкий сбор контактов перед входом в каталог.

## Обязательные экраны TMA для кофейни

```
Главная / витрина
├── Баннер или приветствие
├── Популярные / новинки
└── Кнопка перехода в меню

Каталог
├── Горизонтальные табы категорий (закреплены вверху)
├── Карточки товаров (фото + название + цена + кнопка)
└── Поиск (опционально)

Карточка товара
├── Фото
├── Название + описание
├── Модификаторы (молоко, сироп, размер) — как bottom sheet
└── Кнопка «Добавить в корзину»

Корзина
├── Список позиций с редактированием
├── Итог
├── Поле промокода (опционально)
└── Кнопка оформления (Telegram MainButton)

Оформление заказа
├── Выбор: самовывоз / доставка
├── Временной слот (если самовывоз)
├── Комментарий к заказу
└── Оплата

Статус заказа
├── Прогресс (принят → готовится → готов)
└── Уведомление в чат

Профиль / лояльность (опционально)
├── Баланс бонусов
├── История заказов
└── Реферальная ссылка
```

---

## Обязательные UI-элементы

| Элемент | Почему обязателен |
|---|---|
| Горизонтальные табы категорий | Главная навигация, видна всегда |
| Карточка с фото 4:3 или квадрат | Продаёт визуально, Telegram любит ретину |
| Telegram MainButton | Нативный CTA, Full-width, прилипает к низу |
| Bottom sheet для модификаторов | Не уводит со страницы, ощущается нативно |
| Бейдж корзины в хедере | Постоянная видимость состояния |
| Skeleton-экраны при загрузке | Вместо спиннера — ощущение скорости |
| Haptic feedback при добавлении | Тактильная обратная связь = нативность |
| Сохранение корзины (localStorage) | Пользователь может уйти и вернуться |

---

## Анти-паттерны (что делать нельзя)

- Делать «сайт в iframe» с кучей ссылок и навигацией
- Спиннеры вместо skeleton
- Запрашивать контакты/геолокацию без контекста
- Дублировать навигацию (своя кнопка «назад» + нативная)
- Открывать модификаторы на отдельной странице

---

## Что мы можем сделать лучше

1. **Визуал кофейной атмосферы** — большинство решений выглядят как безликий общепит. Тёплая палитра, крафтовая эстетика.
2. **Кастомизация напитка как ритуал** — не сухой список модификаторов, а живой выбор («как ты любишь?»).
3. **Простота как ценность** — без регистрации, без лишних шагов. Открыл — выбрал — заказал.
4. **Прозрачность** — показывать статус заказа внутри бота, без внешних ссылок.
5. **Личный голос** — тексты живые, как в кофейне у стойки, а не корпоративные.

## Экспертная оценка

> Идея: каталог услуг кофейни как TMA. Клиент открывает бота → нажимает кнопку → видит услуги с ценами и кнопкой «Выбрать».

---

### Взгляд UX-дизайнера

**Типичные ошибки новичков:**

- **«Сайт в iframe»** — верстают как обычный лендинг, пихают хедер, футер, меню сверху. В TMA нет места для лишнего — экран маленький, один главный экран, одно действие.
- **Слишком много шагов до цели** — «нажми кнопку → увидишь приветствие → нажми ещё → выбери категорию → только потом услуги». Каждый лишний шаг — потеря части пользователей.
- **Кнопка «Выбрать» без ответа** — нажал, ничего не произошло. Нет визуального подтверждения. Пользователь нажимает ещё раз.
- **Мелкие зоны касания** — кнопки меньше 44px. На мобиле попасть невозможно, раздражение гарантировано.
- **Текст вместо карточки** — список услуг просто текстом без фото и цены сразу. Глаз не цепляется, решение откладывается.

**Что обязательно должно быть:**

- Услуга видна целиком сразу: фото / иллюстрация + название + цена + кнопка. Никаких «нажми чтобы узнать цену».
- Цена указана всегда. Скрытая цена = потеря доверия.
- После нажатия «Выбрать» — мгновенная реакция: изменение цвета кнопки, haptic-виброотклик, или переход к подтверждению.
- Один ключевой CTA на экран. Не три кнопки «Выбрать», «Подробнее», «В корзину» — только один.
- Если есть прокрутка — визуально намекнуть (часть следующей карточки видна снизу).

**Что бесит пользователей:**

- Долгая загрузка без индикации (спиннер крутится 3+ секунды — закрывают).
- Кнопка которая «зависает» после нажатия без реакции.
- Невозможность вернуться назад без потери выбора.
- Форма с полями там, где достаточно одной кнопки «Написать в Telegram».
- Попап с просьбой оставить контакты до того, как человек увидел хоть одну услугу.

**Рекомендации для нашего проекта:**

- Открытие = сразу каталог. Без экрана-заставки, без «добро пожаловать».
- Каждая карточка услуги: фото/иллюстрация → название → короткое описание (1 строка) → цена → кнопка «Выбрать».
- После «Выбрать» — Telegram MainButton меняется на «Написать в Telegram» или открывает диалог с менеджером.
- Максимум 2 уровня навигации: список категорий → карточки услуг.

### Взгляд маркетолога

**Что заставит записаться:**

- **Социальное доказательство в карточке** — «Уже записались 47 человек» или «★ 4.9 из 52 отзывов» рядом с услугой. Работает лучше любого текста.
- **Ограниченность / срочность** — «Осталось 2 места на эту неделю» или «Акция до пятницы». Даже если это мягкий намёк — конверсия растёт.
- **Первый шаг должен быть бесплатным или очень дешёвым** — по данным tgEDA, 97% кофеен дают подарок или скидку на первый заказ. Это работает: снижает страх первого контакта.
- **Конкретная цена, а не «от ...»** — «от 500 ₽» отпугивает. «500 ₽» — располагает. Неопределённость = тревога = закрыть.
- **Фото реальных работ / результата** — не стоковые картинки. Живые фото создают ощущение настоящего места.

**Элементы, повышающие конверсию:**

| Элемент | Эффект |
|---|---|
| Цена на карточке сразу | Убирает барьер «а вдруг дорого» |
| Кнопка «Выбрать» → диалог с менеджером | Снижает трение до минимума |
| Отзыв прямо под услугой | +15–30% к доверию (типичный benchmark) |
| «Популярное» / «Рекомендуем» бейдж | Направляет выбор, снижает паралич |
| Акция первого визита (скидка / бонус) | Конвертирует нерешительных |
| Быстрый ответ (<5 минут) | Решает больше, чем любой дизайн |

**Что убивает конверсию:**

- Нет цены → «напишите для уточнения» → человек уходит.
- Слишком большой выбор без приоритизации. Если 20 услуг — первые 3–5 должны быть выделены как «лучшие» или «популярные».
- CTA «Подробнее» вместо «Выбрать» — слово «подробнее» ведёт читать, не действовать.
- Отсутствие следующего шага после выбора — пользователь нажал, и... тишина.

**Рекомендации для нашего проекта:**

- Первый экран: 2–3 «хитовые» услуги с ценами и бейджем «Популярно».
- Кнопка «Выбрать» → сразу открывает чат с менеджером с преднаполненным текстом (например: «Хочу записаться на [Название услуги]»).
- Добавить блок с отзывами или хотя бы счётчик записавшихся.
- Акция первого визита — в самом верху или на карточке первой услуги.

### Взгляд Telegram-разработчика

**Ограничения TMA, о которых нужно знать:**

- **Размер экрана непредсказуем** — Mini App открывается в половину экрана, но пользователь может развернуть. Верстать нужно для обоих состояний. Никакого фиксированного `height: 100vh` без `tg.expand()`.
- **iOS и Android ведут себя по-разному** — особенно с клавиатурой (она сжимает viewport на Android, на iOS — нет). Если есть поля ввода — тестировать на обоих.
- **Нет доступа к файловой системе** — нельзя сохранить файл «на устройство» из TMA. PDF-каталог лучше отправлять через бот в чат.
- **Telegram WebApp API — не весь Telegram** — нет прямого доступа к истории чата, контактам, платёжной истории без явного разрешения пользователя.
- **Кэширование — друг и враг** — браузерный движок Telegram агрессивно кэширует. При обновлении контента нужна инвалидация (cache-busting в именах файлов или заголовки `Cache-Control`).
- **Навигация — только одна страница** — TMA это SPA (или статичная страница). Переходы между «страницами» — это показ/скрытие блоков или push-состояния истории. Нельзя просто сделать `<a href="/catalog">`.

**Что работает хорошо:**

- `MainButton` — нативная кнопка Telegram, всегда видна, не конкурирует с остальным UI. Использовать как главный CTA.
- `BackButton` — нативная кнопка «назад» в хедере. Подключить — 2 строки кода. Пользователи ожидают её.
- `HapticFeedback` — виброотклик при касании. На iOS и Android работает нативно. Добавляет ощущение «живого» приложения.
- `themeParams` — TMA автоматически получает цвета темы Telegram пользователя. Используй `var(--tg-theme-bg-color)` и `var(--tg-theme-text-color)` — приложение будет выглядеть нативно в любой теме.
- `openTelegramLink` — открывает ссылку в Telegram (например, чат с менеджером) без выхода из TMA.
- `LocalStorage` — работает, сохраняет данные между сессиями. Хранить выбор пользователя, черновик заявки.

**Что лучше не делать:**

- **Не использовать внешние шрифты через Google Fonts** — загрузка замедляет старт, часть устройств блокирует внешние запросы. Системные шрифты или self-hosted.
- **Не делать анимации на CSS `top/left`** — только `transform` и `opacity`. Иначе — дёрганье на слабых устройствах.
- **Не полагаться на `window.alert()`** — в TMA не работает стандартный браузерный алерт. Только `tg.showAlert()` или кастомный UI.
- **Не открывать внешние ссылки через `window.open()`** — Telegram перехватывает это непредсказуемо. Только `tg.openLink()` или `tg.openTelegramLink()`.
- **Не делать тяжёлый JS-бандл** — TMA загружается каждый раз заново (нет постоянного жизненного цикла как у нативного приложения). Бандл > 200kb ощущается медленно.
- **Не хранить чувствительные данные в LocalStorage** — это клиентское хранилище, доступное коду страницы. Секреты — только на сервере.

**Рекомендации для нашего проекта:**

- Стек: HTML + CSS + vanilla JS или лёгкий фреймворк (Preact, Svelte). Без тяжёлых React-приложений для такого объёма.
- При нажатии «Выбрать»: `tg.MainButton.show()` с текстом «Записаться» → клик по MainButton → `tg.openTelegramLink()` на чат менеджера с преднаполненным текстом.
- Поддержка темы: использовать CSS-переменные `--tg-theme-*` для фона и текста, поверх накладывать фирменные цвета кофейни только для акцентов.
- Инициализация: `tg.expand()` сразу при открытии — разворачивать на весь экран.
- `tg.BackButton.show()` на всех экранах кроме главного.

---

### Сводный список рекомендаций

**Делаем обязательно:**
1. Открытие = сразу каталог, без экранов-заставок
2. Каждая карточка: фото + название + цена + кнопка «Выбрать»
3. «Выбрать» → `openTelegramLink` на чат менеджера с преднаполненным текстом заявки
4. `tg.expand()` при старте — полноэкранный режим
5. `MainButton` как главный CTA
6. `BackButton` на внутренних экранах
7. `HapticFeedback` при интерактиве
8. Цена — всегда конкретная, не «от ...»
9. 2–3 «популярных» услуги на первом экране с бейджем
10. CSS-переменные темы Telegram + фирменные акценты кофейни

**Делаем по возможности:**
- Счётчик записавшихся или отзыв под услугой
- Акция первого визита на видном месте
- Skeleton-загрузка вместо спиннера
- Сохранение выбора в localStorage

**Не делаем:**
- Форм с полями там, где хватает кнопки
- Запроса контактов до показа каталога
- Google Fonts и внешних тяжёлых библиотек
- `window.alert()`, `window.open()`
- Скрытых цен и кнопок «Подробнее» вместо «Выбрать»
- Больше одного CTA на экране
